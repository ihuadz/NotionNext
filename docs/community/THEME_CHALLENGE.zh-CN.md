# NotionNext 主题共创活动

NotionNext 已经内置多种主题，但不同站点仍然会需要更多视觉风格：个人博客、作品集、知识库、杂志、产品页、导航站、极简主页、日系卡片、Material 风格、复古博客等。

这个长期活动面向所有正在使用、准备使用或喜欢折腾主题的读者。即使暂时不会完整开发一个主题，也可以通过参考站点、截图、使用场景、文档补充、AI 辅助改造和小范围 PR 参与进来。

## 可以参与什么

| 参与方式 | 适合人群 | 产出 |
| --- | --- | --- |
| 提交主题灵感 | 喜欢收集好看站点、正在找主题方向的使用者 | 参考链接、截图、风格描述 |
| 补充使用场景 | 已经部署 NotionNext 的站长 | 哪类站点适合某个主题、有哪些痛点 |
| 整理 UI 拆解 | 设计参与者、审美向使用者 | 色彩、布局、卡片、导航、移动端说明 |
| AI 辅助改造 | 想学习主题开发的使用者 | 一个组件、一个页面、一个主题雏形 |
| 补文档和截图 | 不想碰代码的参与者 | 主题说明、配置示例、预览图 |
| 提交 Draft PR | 已能本地运行项目的贡献者 | 可被社区继续接力的小改动 |

## 从哪里开始

1. 先看现有主题：[主题全览](../user-guide/themes/THEMES_CATALOG.md)。
2. 选择一个喜欢但还不完全满足需求的主题。
3. 准备 1 到 3 个参考站点或截图。
4. 在 [GitHub Discussions](https://github.com/notionnext-org/NotionNext/discussions) 发起主题建议。
5. 如果准备动手改造，再阅读 [AI 主题开发提示词](../user-guide/development/ai-theme-prompts.md) 和 [AI 辅助开发教程](../user-guide/development/notion-next-develop-with-ai.md)。

不需要一开始就做完整主题。一个按钮样式、一组卡片布局、一个首页区块、一个移动端导航，都可以成为有效贡献。

## 推荐发帖格式

```markdown
## 想要的主题风格

- 风格关键词：
- 适合什么站点：
- 参考链接：
- 截图或说明：

## 希望解决的问题

- 现在使用的主题：
- 不满意的地方：
- 希望新增或调整的页面：

## 可以参与的部分

- [ ] 只提供参考资料
- [ ] 可以补充截图和文档
- [ ] 可以尝试 AI 辅助修改
- [ ] 可以开 Draft PR
```

## 适合新参与者的小任务

- 给一个主题补充更清楚的适用场景。
- 给一个主题补充桌面端和移动端截图。
- 整理某种风格的 3 个参考站点。
- 把一个主题建议拆成更具体的页面或组件需求。
- 用 AI 改一个主题的小组件，例如按钮、卡片、文章列表或导航。
- 给已有主题补充深色模式、移动端问题或配置说明。

这些任务都适合作为 `good first issue` 或 Draft PR。目标不是一次做完，而是让想法可以被社区继续接力。

## AI 辅助开发路径

建议按小步骤推进：

1. 选一个基准主题，例如 `example`、`simple`、`starter`、`endspace`。
2. 用截图或参考链接让 AI 拆解页面结构。
3. 只改一个页面或一个组件。
4. 本地运行，检查桌面端、移动端和深色模式。
5. 补充主题说明、预览图和改动范围。
6. 开 Draft PR，让社区继续 Review 和完善。

更完整的主题迁移与贡献要求见 [主题迁移指南](../developer/THEME_MIGRATION_GUIDE.zh-CN.md)。

## 什么样的建议更容易被采用

更容易推进：

- 有具体参考链接或截图。
- 能说明适合哪类站点。
- 能基于现有主题小步改造。
- 改动范围清楚，例如“首页卡片布局”或“移动端导航”。
- 愿意接受社区继续二次修改。

较难推进：

- 只有一句“想要更好看”。
- 需要一次性重写多个主题。
- 只服务单个私人站点，无法抽象成通用能力。
- 没有截图、参考站点或验收标准。

## 成果会去哪里

成熟的主题贡献通常会沉淀到这些地方：

- GitHub Discussions：主题方向、参考资料、共创讨论。
- GitHub Issues：可认领任务、Bug、配置问题。
- Draft PR：主题雏形、小组件、文档和截图。
- 文档站：主题使用说明、AI 开发教程、常见问题。
- 主题目录：稳定后进入主题全览和主题切换清单。

## 长期运行方式

活动会长期开放，不按一次性比赛结束。社区可以随时提交灵感、认领小任务、补文档或开 Draft PR。

为了减少单个维护者压力，日常运转会尽量依赖公开讨论、任务池、标签、模板和社区互助。完整运营分工见 [主题生态长期运营模型](./THEME_ECOSYSTEM_OPERATING_MODEL.zh-CN.md)。
